# Glass House Project Mapping

Created: 2026-05-06
Purpose: Convert the glass house metaphor into practical project decisions.

## Immediate Project Meaning

The glass house idea gives Foundation a clearer identity:

Foundation / Build Your Own Scientist is a system for watching what happens when human knowledge, AI reasoning, memory, tools, and pressure pass through the same structure.

It should not try to become everything.

It should become excellent at:

- seeing distortion
- catching stale truth
- finding contradictions
- noticing pressure buildup
- requiring evidence before action
- separating public truth from private/sensitive material
- preserving beautiful ideas without overclaiming them

## How It Applies To Current Projects

### Foundation / Build Your Own Scientist

Role: the glass-house inspection system.

Priority:
- stabilize
- simplify
- inspect
- gate
- document current truth

Do not add new feature wings until the structure is reliable.

### Foundation App / Liquid Workbench

Role: the visible rooms of the house.

Priority:
- one canonical entry point
- fewer visual artifacts
- less UI identity confusion
- clear workflow
- no generated clutter in source control

The app should help the user see what is happening, not become another source of glare.

### Geometry-Likelihood

Role: a case study in light correction.

The project originally caught a beautiful signal, then later evidence changed the conclusion.

The right response was not embarrassment or deletion.

The right response was to update the public truth:

- early positive signal becomes historical
- formal null result becomes current
- falsification becomes the value

This is exactly what Foundation should help with.

### CCS Patent Project

Role: a sealed room.

CCS may be conceptually related to the same safety/control philosophy, but it must remain separate while trustee/legal authority is unresolved.

Foundation can track the existence of the sealed room.

Foundation should not copy its contents.

### Aegis / CurrentTruth / Safety Gate

Role: future extracted architecture.

These are not separate builds yet. They are future structural systems that may emerge once Foundation is stable:

- CurrentTruth: what is true right now?
- Safety Gate: should the system proceed, hold, or ask for review?
- Aegis: can the system notice drift, instability, or unsafe pressure before damage occurs?

## Design Questions For Future Work

Use these questions when deciding what to build next:

- Does this make the house easier to inspect?
- Does this reduce glare or add glare?
- Does this help us tell warmth from fire?
- Does this separate public truth from private/sensitive material?
- Does this make current truth clearer?
- Does this reduce project scatter?
- Does this require evidence before action?
- Does this preserve beauty without turning it into overclaim?

If the answer is no, do not build it yet.
