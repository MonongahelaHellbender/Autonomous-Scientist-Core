# Foundation Visual Identity Direction

Created: 2026-05-06

## Core Direction

Foundation should feel bright, clean, futuristic, cinematic, and real.

The interface should look like a serious sci-fi control room that could exist:

- luminous
- technical
- calm
- structured
- high signal
- not cartoonish
- not mystical as the main interface
- not a novelty skin

## Brain Representation Rule

The overall application shell stays stable.

The brains may adapt their own visual representation as they evolve.

This means:

- the app remains coherent and usable
- the brain forms can change with training state
- representation is symbolic, not a claim of consciousness
- visual evolution is tied to saved artifacts and system behavior
- the user can inspect why a form was selected

## Current Representation Modes

The current modes are:

- genesis: forming signal core
- prism: command and orchestration
- orbital: full-cycle execution
- lattice: latent analysis
- archive: evidence memory
- halo: synthesis and convergence
- flux: adaptive loop behavior
- crystalline: experiment chamber
- sentinel: safety and guardrail state
- ledger: learning memory

## Design Boundary

Do:

- let brains represent themselves through technical forms
- keep every representation inspectable
- connect visual changes to actual state
- preserve bright cinematic realism

Do not:

- imply the system is conscious
- make the whole app visually unstable
- hide controls behind decorative effects
- make CCS/Aegis visible as a public/default brain
- use patent-sensitive representations without a private boundary

## Best Description

Foundation is the clean cinematic environment.

The brains are the adaptive signal forms inside it.
